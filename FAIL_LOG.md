# FAIL LOG

コーディング AI / 開発者用の失敗台帳。**同じ失敗を繰り返さないため**、詰まったら必ず追記する。
書式: 日付 / 症状 / 試したこと / 根本原因 / 解決策。

---

## テンプレート
- **日付**:
- **症状**: （何がどう失敗したか。エラーメッセージも）
- **試したこと**: （順に）
- **根本原因**: （なぜ起きたか）
- **解決策**: （最終的にどう直したか。再発防止策）

---

## 既知の落とし穴（先に読むこと）
- **`file://` で開くと動かない**: ES Modules と OAuth の制約。必ず http(s) で配信する（`python3 -m http.server`）。
- **Drive の OAuth エラー `redirect_uri_mismatch` / `origin` 系**: OAuth クライアント ID の「承認済み JavaScript 生成元」に配信元 URL を登録する。
- **CORS でゲートウェイに繋がらない**: 送信先 API がブラウザ直アクセスを許可していない場合はプロキシが必要。
- **CryptoKey を IndexedDB に保存できない**: 構造化複製で保存可能。`extractable:false` のまま put すること（rawキーを取り出そうとしない）。
- **ストリームが表示されない**: エンドポイントが `text/event-stream` を返しているか確認。返さない場合は JSON フォールバックが動く。
