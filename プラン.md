## 1. 目的

ChatGPT / Gemini のようなチャット UI を持ち、**OpenAI 互換エンドポイント**
(`POST /v1/chat/completions`) に接続する、**ビルド不要の静的フロントエンド**。
`app.config.js` と `styles/theme.css` の差し替えで各種アプリに転用できる基盤にする。

- サーバー実装なし（利用者自身の API キー / エンドポイントを使う）。
- ブラウザだけで動く。ES Modules、ビルド不要、静的配信でそのまま動作。

## 2. 全要件（確定版・最終形）

今回作らないものも含め、目指す最終形の要件を全て列挙する。
`[MVP]` = 今回実装 / `[後]` = 後続マイルストーンで追加。

### 2.1 チャット UI
- `[MVP]` サイドバー：会話履歴の一覧 / 新規会話 / 会話の選択・削除・リネーム。
- `[MVP]` メイン：user/assistant の吹き出し表示、Markdown とコードの整形、コードのコピー。
- `[MVP]` 入力欄：複数行、送信 / 停止（生成中断）。
- `[MVP]` ストリーミング逐次描画（`stream:true`、非対応時は非ストリームにフォールバック）。
- `[MVP]` レスポンシブ（モバイルはドロワー、PC は固定サイドバー）、フォーカス可視。
- `[後]` 画像添付・貼り付け・カメラ撮影（`supportsImages` なモデル向け）。
- `[後]` 空チャットのスターター例、`prefers-reduced-motion` 尊重。

### 2.2 接続先（プロバイダ）管理
- `[MVP]` 項目：名前 / `endpoint`(baseURL) / 認証方式 / API キー。
- `[MVP]` 認証方式：`bearer`（`Authorization: Bearer <key>`・既定）/ `raw`（キー直入れ）/ `custom`（ヘッダ名指定）。
- `[後]` 複数プロバイダの登録・切替。

### 2.3 モデル管理
- `[MVP]` 単一モデル文字列を設定（`model`）。会話はこのモデルで送る。
- `[後]` モデルの複数登録・追加/編集/削除、会話ごとの切替、`supportsImages`・既定生成パラメータ。

### 2.4 チャットログ保存
- `[MVP]` 会話 = メッセージ配列＋メタ（タイトル・作成/更新時刻・使用モデル）。
- `[MVP]` 自動保存・復元・削除・リネーム。タイトルは初回メッセージから自動生成（編集可）。
- `[MVP]` 保存先はブラウザの `localStorage`（MVP は暗号化なし・プレーン）。
- `[後]` JSON エクスポート / インポート。

### 2.5 システムプロンプト（タイトル付き）
- `[後]` 「タイトル＋本文」プリセットを複数保存、会話へ適用、既定指定、編集・削除。

### 2.6 アーティファクト保存
- `[後]` アシスタント出力からコード/HTML/文書を抽出して独立保存、一覧・プレビュー・エクスポート。

### 2.7 暗号化保存
- `[後]` AES-GCM ＋ **非抽出(non-extractable) CryptoKey を IndexedDB に保持**して自動解錠。
  保存されるのは暗号文・IV のみ。任意でパスフレーズ（PBKDF2）モード。
- `[後]` API キーも暗号化対象。
- ※ MVP は簡潔さ優先で暗号化を入れない。§5 に残存リスクを明記。

### 2.8 保存先の選択（ローカル / Google ドライブ）
- `[後]` ストレージ抽象化 + 2 実装（`local` / `gdrive`）を切替。
- `[後]` Google ドライブは利用者自身の OAuth クライアント ID を使い、`appDataFolder` に暗号化 vault を保存。

### 2.9 ガードレール
- `[後]` フック地点：送信前（入力）と受信後（出力）。有効/無効トグルとルール保持の器。既定 no-op。

### 2.10 転用性（基盤としての要件）
- `[MVP]` 触るのは主に `app.config.js` と `styles/theme.css` のみ、を目標にする。
- `[MVP]` Core（API 呼び出し・保存）と UI を分離しておく。

### 2.11 非機能
- `[MVP]` Markdown は安全レンダリング（生 HTML を実行しない / エスケープしてから整形）。
- `[MVP]` 静的配信で動作・ビルドなし。最新 Chrome/Edge/Safari/Firefox・モバイル対応。
- `[MVP]` エラーは分かりやすく（401→トークン、404→パス、CORS 等の指針）。
- `[後]` PWA（インストール・オフライン起動）。※ SW キャッシュで詰まった経緯があり、コア安定後に再導入。

## 3. 今回のMVPスコープ（これだけ作る）

**チャットのコアのみ。** 具体的に：

1. **設定画面**：`endpoint` / `apiKey` / 認証方式 / `model` の 4 項目を入力・保存。初回は自動で開く。
2. **チャット送受信**：`POST {endpoint}` に OpenAI 互換 JSON を送り、応答を表示。
3. **ストリーミング**：SSE（`text/event-stream`）を逐次描画。非対応なら通常 JSON にフォールバック。生成停止ボタン。
4. **会話履歴**：`localStorage` に会話を保存・復元・選択・削除・リネーム。タイトル自動生成。
5. **Markdown / コード整形**：安全レンダリング、コードブロックのコピー。
6. **レスポンシブ**：モバイル=ドロワー、PC=固定サイドバー。

MVP に**含めない**：暗号化・Google ドライブ・アーティファクト・システムプロンプト・
複数モデル/プロバイダ切替・画像/カメラ・Web検索・ガードレール・PWA。（すべて §2 に要件として記録済み）

## 4. アーキテクチャ（シンプル構成）

UI と Core（API/保存）を薄く分離。ファイルは最小限。

```
index.html            … 画面骨格（サイドバー / メイン / コンポーザ / 設定モーダル）
app.config.js         … アプリ名・入力例プレースホルダ・既定値（転用時に触る）
styles/theme.css      … 配色・タイポ（:root トークン差し替えでテーマ変更）
src/
  api.js              … OpenAI 互換クライアント（chat completions・ストリーミング・停止）
  store.js            … localStorage の会話/設定 CRUD（メモリ状態 + 永続化）
  markdown.js         … 安全な軽量 Markdown → HTML（エスケープ→整形）
  app.js              … 画面描画とイベント配線（store と api を繋ぐ）
```

データフロー：
- 送信：UI → `store` に user メッセージ追加 → `api.chat()` で送信 → ストリームを assistant メッセージに反映 → `store.persist()`。
- 読込：起動時 `store.load()` → 会話一覧描画 → 選択で本文描画。

## 5. 保存とセキュリティ（MVP の割り切り）

- MVP は `localStorage` にプレーン保存する。**API キーも平文で入る**。
  同一ブラウザにアクセスできる者は読める。これは MVP の割り切りで、
  §2.7 の暗号化（非抽出鍵 AES-GCM）を後続で導入して解消する。
- 第三者送信はしない。送信先は利用者が設定した `endpoint` のみ。
- Markdown は生 HTML を実行しない（先にエスケープしてから整形）。

## 6. データモデル（論理スキーマ）

```
Settings   { endpoint, apiKey, authMode('bearer'|'raw'|'custom'), customHeader?, model }
Conversation { id, title, model, createdAt, updatedAt, messages: Message[] }
Message      { role('user'|'assistant'|'system'), content: string, createdAt }
```
- `schemaVersion` を持たせ将来のマイグレーションに備える。

## 7. ロードマップ（MVP 後の段階追加）

- **M1（本MVP）**：チャットコア（設定 / 送受信 / ストリーミング / 履歴 / Markdown）。
- **M2**：暗号化保存（非抽出鍵 AES-GCM / IndexedDB へ移行、API キー暗号化）。
- **M3**：システムプロンプト（タイトル付き）＋アーティファクト抽出・一覧。
- **M4**：複数プロバイダ/モデル管理・切替、画像添付・カメラ。
- **M5**：Google ドライブ（appDataFolder）保存先＋切替、JSON エクスポート/インポート。
- **M6**：ガードレール枠（no-op）＋ PWA 再導入。

## 8. 意思決定ログ

- 全部盛りで詰まったため、**動くコアから積み上げる**方針に変更（このファイルが起点）。
- MVP はビルド不要 ES Modules。保存は `localStorage`（暗号化・IndexedDB は M2 で導入）。
- 認証は既定 `bearer`。接続先ごとに `raw`/`custom` へ変更可。
- ストリーミングは SSE、非対応エンドポイントは通常 JSON にフォールバック。
- Markdown は軽量自作（安全レンダリング優先）。テーブル等は将来拡張。
